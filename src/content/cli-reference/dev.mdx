# gofasta dev

Starts the development server with automatic hot reload powered by [Air](https://github.com/air-verse/air). When you change any `.go` file, the server automatically rebuilds and restarts, giving you a fast feedback loop during development.

## Usage

```bash
gofasta dev
```

Run this command from the root directory of your Gofasta project.

This command takes no flags.

## Examples

Start the dev server:

```bash
gofasta dev
```

## How It Works

The `gofasta dev` command performs the following steps:

1. **Run migrations** -- if a database is available, pending migrations are applied automatically
2. **Print endpoint URLs** -- displays the URLs for REST API, GraphQL, Swagger UI, and other endpoints
3. **Start Air** -- runs `go tool air` to watch for file changes and hot reload the server

The Air configuration is stored in `.air.toml` at the project root. You can customize watched directories, excluded paths, build commands, and delay settings by editing this file.

## Configuration

The port is determined by (in order of precedence):

1. `GOFASTA_` prefixed environment variables
2. `PORT` environment variable
3. `config.yaml` settings
4. Default: `8080`

## Available Endpoints

Once the dev server is running, the following endpoints are available:

| Endpoint | URL |
|----------|-----|
| REST API | `http://localhost:8080/api/v1/` |
| GraphQL | `http://localhost:8080/graphql` |
| GraphQL Playground | `http://localhost:8080/graphql-playground` |
| Health Check | `http://localhost:8080/health` |
| Swagger UI | `http://localhost:8080/swagger/index.html` |

Replace `8080` with your configured port if different.

## Using Docker

If you prefer running inside Docker with hot reload, use the provided Makefile command instead:

```bash
make dev
```

This starts the Docker Compose development profile, which includes both the application with Air hot reload and the database service.

## Troubleshooting

**Port already in use** -- If you see `bind: address already in use`, another process is using the port. Either stop that process or change the port in `config.yaml` or via the `PORT` environment variable.

**Air not found** -- The `gofasta dev` command uses `go tool air` internally. If you encounter issues, make sure you have run `gofasta init` to install all dependencies.
